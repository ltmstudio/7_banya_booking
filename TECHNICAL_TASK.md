Техническое задание (ТЗ) — Финальная версия 2.0 Система бронирования для банного комплекса Дата: 03.03.2026 Стек: Laravel 11 + Vue 3 (Composition API + Pinia), MySQL/PostgreSQL, полностью responsive (mobile-first) Языки системы: Русский + Туркменский (полная локализация через Laravel Localization + Vue i18n, переключатель языка на сайте и в админке)
________________________________________
1. Общая архитектура
•	Публичный сайт — лендинг + модуль онлайн-бронирования.
•	Админ-панель — дашборд, управление, отчёты, касса.
•	Одна локация, несколько банных комнат (не филиальная система).
•	Полная мобильная адаптация всех экранов.
2. Роли и права доступа
•	Админ — полный доступ.
•	Менеджер — всё кроме удаления записей.
•	Кассир — просмотр заявок, смена статуса на «Оплачено», работа с кассой и печать чеков. Не может удалять и управлять пользователями.
3. Публичный сайт
Страницы:
•	Главная
•	О нас
•	Услуги (комнаты + доп.услуги)
Модуль бронирования:
•	Выбор комнаты (с фото, категорией, ценой)
•	Дата, время, длительность (1 / 2 / 3 часа, хоть 10 часов)
•	Количество человек
•	Дополнительные услуги (множественный выбор с иконками)
•	Контакты: Имя (обяз.), Телефон (обяз.), Фамилия и Дата рождения — не обязательны. В день ДР – иконка тортика
Ценообразование на сайте:
•	Базовая цена за 1 час (из справочника).
•	Акционная цена за 1 час (отдельное поле для стимуляции онлайн-оплаты).
•	При акции: зачёркнутая старая цена + новая фиксированная цена (пример: 3 часа — 900 TMT → 750 TMT).
Попап-акция (дисклеймер):
•	Редактируется в админке (заголовок + текст).
•	Показывается при первом заходе на сайт (можно закрыть навсегда).
Визуальные индикаторы комнат:
•	Свободна → зелёная галочка
•	Занята → красный крестик
•	Период уборки → иконка метёлки
Фото и иконки:
•	Банные комнаты — полноразмерные фото.
•	Дополнительные услуги — только иконки (SVG/PNG).
4. Админ-панель
4.1. Главный дашборд (экран после входа)
•	Заявки на сегодня (таблица + мини-календарь).
•	Клиенты, у которых сегодня День рождения (с иконкой 🎂).
•	Быстрые действия.
4.2. Справочники
Банные комнаты
•	Название
•	Категория (Семейная / VIP / Обычная)
•	Описание
•	Фото (несколько)
•	Базовая цена за 1 час
•	Акционная (онлайн) цена за 1 час
•	Статус (активна / отключена)
•	При отключении — предупреждение + опция отмены всех активных броней.
Дополнительные услуги
•	Название
•	Иконка (загрузка SVG/PNG)
•	Описание
•	Цена
•	Статус
4.3. Клиенты
•	Имя, Фамилия, Телефон (уникальное поле)
•	Дата рождения (заполняется при приходе клиента в заведение)
4.4. Заявки и бронирования
Статусы (по порядку):
1.	Новая (с сайта)
2.	Предварительная бронь
3.	Подтверждено
4.	Оплачено
5.	Завершено (автоматически после окончания слота + буфер)
6.	Отменено (обязательная причин надо указать)
Ключевые механики:
•	Буфер на уборку — настраивается в настройках системы (по умолчанию 15 мин, можно 30 мин). Пример: бронь 11:00–13:00 → следующая возможна с 13:15. Период уборки показывается иконкой метёлки.
•	В карточке заявки менеджер/кассир может поставить любую фиксированную итоговую сумму (ручная скидка без процентов), то есть сохранится та первая сумма и новая сумма(скидка или штраф наооборот)
•	При будущей онлайн-оплате автоматически применяется акционная цена комнаты.
Касса
•	Интеграция с фискальным регистратором (Атол / Штрих-М и т.п.).
•	При смене статуса на «Оплачено» — автоматическая печать чека.
Онлайн-оплата (placeholder)
•	Полностью предусмотрено место для интеграции с банком «Рысгал» (Туркменистан).
•	На сайте будет кнопка «Оплатить онлайн» (пока неактивна).
•	После успешной оплаты статус автоматически меняется на «Оплачено».
4.5. Единый Audit Log (журнал действий)
•	Видит только Админ.
•	Фиксирует ВСЁ: кто, когда, какое поле, старое/новое значение.
•	Один общий журнал для всех сущностей.
4.6. Отчёты (только Админ)
•	Заработок за любой период (день / неделя / месяц / произвольный диапазон).
•	Разбивка:
o	По каждой комнате (выбирается фильтром)
o	Общий заработок со всех комнат
o	Отдельно — только дополнительные услуги
•	Экспорт в Excel.
5. Дополнительные требования
•	Полная двухязычность (русский + туркменский).
•	SMS-уведомления — отмечено как будущая функция, сейчас не надо (placeholder в коде).
•	Все формы и таблицы адаптивны.
